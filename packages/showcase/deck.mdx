# Root Frontmatter

Deck-level settings live in the first frontmatter block:

```yaml
---
title: "Honeydeck Feature Showcase"
description: "A structured showcase deck."

defaultLayout: Default
aspectRatio: "16:9"
colorMode: system
pdfSteps: all

```

- `layouts` loads a project-local layout map.
- `defaultLayout` selects slide layout when no slide-level `layout` is set.
- `showSlideNumbers` enables the global slide-number overlay.
- `pdfSteps: all` makes PDF export include step-by-step pages by default.

# Syntax Highlighting 💻

Honeydeck highlights code at compile time with Shiki:

```typescript
import { createHighlighter } from 'shiki'

const highlighter = await createHighlighter({
  themes: ['github-light', 'github-dark'],
  langs: ['typescript'],
})

const html = highlighter.codeToHtml(code, { lang: 'typescript' })
```

---

# Plain Fallback Code

Unsupported or unknown languages still render as readable code:

```madeuplang

  keep content visible
  use Honeydeck code tokens
end
```

# Timeline Rules

- Slide timeline starts at `stepIndex = 0`.
- First reveal/custom timeline entry activates at step 1.
- Code block first highlight group is the baseline and consumes no step.
- Later code groups consume one step each.
- Nested timeline producers flatten in document order.
